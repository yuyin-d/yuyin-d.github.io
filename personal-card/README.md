# 个人数字名片 - 玉印

## 作品简介

这是一个现代化的个人数字名片网页，采用赛博朋克（Cyberpunk）风格设计，展示了玉印同学的个人基本信息、技能特长、兴趣爱好以及联系方式。页面通过丰富的视觉效果和交互动画，营造出科技感十足的视觉体验，非常适合作为程序员的个人展示页面。

## 页面主要内容说明

页面以卡片式布局呈现，包含以下主要模块：

**头像区域**：顶部展示个人头像，采用圆形设计，带有发光效果和旋转动画，鼠标悬停时会放大并旋转。

**基本信息**：展示姓名、学号、专业，文字采用霓虹灯效果，具有强烈的视觉冲击力。

**个人签名**：展示座右铭「人生苦短,我学Python」，使用紫色霓虹灯风格，上下带有虚线装饰。

**兴趣标签**：包含三个兴趣标签（📚阅读、🏀篮球、🎵音乐），采用青色渐变背景，带有悬停发光效果。

**技能标签**：展示技术栈（HTML、CSS、JavaScript），采用紫色渐变背景，带有扫光动画效果。

**关于我**：详细的个人介绍文字，采用浅色文字在深色背景上显示，左侧带有渐变色条装饰，标题处有脉冲动画。

**联系方式**：展示邮箱和电话，鼠标悬停时文字会向右滑动并变色。

**外部链接**：底部设有"访问我的GitHub主页"按钮，带有渐变背景和悬停发光动画效果。

## 文件结构说明

项目采用标准的静态网页结构，文件组织清晰：

```
personal-card/
├── index.html          # 主页面文件，包含HTML结构
├── style.css           # 样式文件，包含所有CSS样式和动画
├── README.md           # 项目说明文档（本文件）
└── images/
    └── avatar.jpg      # 个人头像图片
```

- **index.html**：采用HTML5语义化标签编写，结构清晰，所有内容通过`div.card`容器包裹，分为`header`（头部）和`main`（主体）两个主要部分。

- **style.css**：包含约450行CSS代码，实现了复杂的视觉效果：
  - 使用CSS3动画实现背景移动、边框发光、标签扫光等效果
  - 采用backdrop-filter实现毛玻璃效果
  - 使用gradient渐变和box-shadow营造霓虹灯发光效果
  - 响应式设计，适配不同屏幕尺寸
  - 自定义滚动条样式

- **images/avatar.jpg**：个人头像图片，建议尺寸为180x180像素或更高分辨率。

## 页面公开访问方式或说明

本作品为纯静态网页，可通过以下方式访问：

1. **本地访问**：直接双击`index.html`文件在浏览器中打开即可查看。

2. **本地服务器访问**：使用VS Code的Live Server插件、Python的http.server、Node.js的http-server等工具启动本地服务器后访问。

3. **线上部署**：可将项目上传到GitHub Pages、Vercel、Netlify等静态托管平台，即可通过互联网公开访问。

页面兼容主流现代浏览器（Chrome、Firefox、Safari、Edge等），推荐使用Chrome或Edge以获得最佳视觉效果。

## 本作业中你认为最能体现"软件作品"而不只是"单个页面"的一点

本作品最体现"软件作品"特质的是**模块化设计思想与可扩展性**。

虽然从表面上看这是一个单页面应用，但其代码结构体现了软件工程的核心思想：

1. **关注点分离**：将HTML结构、CSS样式完全分离到独立文件中，便于维护和修改。这种分层设计是专业软件项目的基本特征，区别于简单地将所有代码写在一个文件中的"单个页面"。

2. **组件化思维**：CSS采用了面向对象的写法，将`.tag`作为基础组件，通过`.interest`和`.skill`两个修饰类实现不同的样式变体。这种组件化思想类似于现代前端框架的设计理念。

3. **可复用性设计**：标签组件的样式设计具有高度可复用性，可以轻松添加新的标签而不需要修改核心样式代码。

4. **用户体验导向**：通过大量的微交互提升用户体验，这种对细节的打磨体现了软件产品的思维，而非仅仅是页面的静态展示。

5. **响应式设计**：考虑了不同设备屏幕的适配，这是专业软件项目必须考虑的问题。

这种工程化的思维方式和对代码质量的关注，使得这个作品不仅仅是一个"页面"，而是一个具有专业水准的"软件作品"。

## AI辅助开发说明

### 是否使用AI

是的，本项目使用了AI辅助开发。

### 使用了哪些工具

主要使用了以下AI工具：

1. **deepseek**：查询了作业流程以及Git的使用方式。

2. **代码智能提示和补全**：使用了IDE自带的AI代码补全功能，提高了编写效率。

3. **WorkBuddy**：作为主要的AI助手，提供了代码编写、样式设计、文件管理等全方位支持。

### 关键提示词

以下是开发过程中使用的几条关键提示词：

1. "帮我创建一个赛博朋克风格的个人数字名片页面，包含头像、姓名、学号、专业、兴趣爱好标签、技能标签、自我介绍和联系方式，使用霓虹灯发光效果和动画。"

2. "为这个页面添加动态渐变背景和网格背景，使用CSS3动画实现，营造科技感。"

3. "优化卡片的视觉效果，添加毛玻璃效果、边框发光动画、悬停交互效果。"

4. "为标签组件添加扫光动画效果，使用CSS伪元素实现。"

### AI主要帮助了哪些部分

AI在以下几个方面提供了主要帮助：

1. **视觉设计构思**：AI提供了赛博朋克风格的配色方案和设计元素建议，包括霓虹灯发光效果、毛玻璃效果等。

2. **CSS动画实现**：复杂的CSS3动画由AI编写，包括关键帧动画的实现细节。

3. **交互效果设计**：鼠标悬停时的各种交互效果主要由AI实现。

4. **代码结构优化**：AI帮助优化了HTML结构和CSS代码组织，使其更加模块化和可维护。

5. **细节打磨**：滚动条样式、文字阴影、边框渐变等细节效果由AI提供实现方案。

### 你对AI结果做了哪些修改

虽然AI生成了大部分代码，但我对结果进行了以下修改和完善：

1. **个人信息调整**：将AI生成的示例数据替换为真实的个人信息。

2. **内容本地化**：将部分英文内容翻译为中文，使其更符合中文用户的使用习惯。

3. **颜色微调**：对AI生成的配色方案进行了微调，降低了部分颜色的饱和度，避免视觉疲劳。

4. **动画参数优化**：调整了部分动画的时长，使动画节奏更加舒缓。

5. **代码注释**：为关键代码段添加了中文注释，便于理解和维护。

6. **响应式优化**：针对移动端显示效果进行了针对性优化，调整了padding和字体大小。

### 你认为AI最不可靠的一点

AI最不可可靠的一点是**对具体业务场景的理解和需求把握**。

在开发过程中，我发现AI存在以下问题：

1. **过度设计倾向**：AI倾向于生成过于复杂的视觉效果和动画，有时会超出实际需求，导致代码冗余和性能问题。例如，AI最初生成的动画效果过多，影响页面加载速度，需要我手动精简。

2. **缺乏上下文理解**：AI无法真正理解项目的具体使用场景和用户群体。例如，它不知道这是一个学生作业，可能会生成企业级产品才需要的复杂功能。

3. **创新与实用的平衡**：AI更注重视觉上的创新和炫技效果，但有时会忽略实用性。比如，某些动画效果虽然好看，但可能会分散用户对核心信息的注意力。

4. **技术选型的合理性**：AI有时会选择过于复杂的技术方案来实现简单的效果，需要根据实际项目情况进行简化。

因此，在使用AI辅助开发时，开发者需要保持批判性思维，不能完全依赖AI的输出，需要结合实际需求进行判断和调整。AI更像是一个"灵感助手"而非"全能解决方案"，最终的产品质量和用户体验仍然需要开发者的专业把控。
